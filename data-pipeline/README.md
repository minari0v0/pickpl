# 🚀 PickPl Data Pipeline Engine

> **네이버 플레이스의 진짜 리뷰 수집부터 Gemini AI의 감성 분석, 백엔드 벌크 적재까지 한 번에 해결하는 3단계 데이터 파이프라인 모듈이에요.**

---

## 💡 PickPl 데이터 파이프라인은 이렇게 작동해요

우리가 가보고 싶은 공간을 찾을 때, 광고성 글이나 뻔한 정보들 때문에 피로했던 적이 있으셨나요? 
PickPl 데이터 파이프라인은 네이버 플레이스에서 진짜 방문자들이 남긴 생생한 리뷰 데이터만 정밀하게 긁어와요. 

그리고 수집에만 그치지 않고, 강력한 **Gemini AI**를 활용해 공간의 진짜 매력(감성, 무드, 어울리는 카테고리)을 똑똑하게 분석한 다음, Spring Boot 백엔드 데이터베이스에 **벌크 적재(Bulk Load)**까지 막힘없이 수행하는 3단계 데이터 엔지니어링 엔진이랍니다. 

이 파이프라인 덕분에 PickPl 서비스는 언제나 신뢰할 수 있고 감성 가득한 장소 정보들로 풍성하게 채워질 수 있어요.

## 🛠️ 개발 환경 설정 및 설치

PickPl 데이터 파이프라인은 Python 3.8+ 이상 환경과 Playwright 브라우저 기반으로 작동해요. 아래의 단계를 따라 개발 환경을 쉽고 빠르게 준비해 보세요.

### 1. 가상환경 활성화 및 패키지 설치
```bash
# 가상환경 활성화 (Windows)
.venv\Scripts\activate

# 의존성 라이브러리 설치
pip install -r requirements.txt

# Playwright Chromium 브라우저 설치
playwright install chromium
```

### 2. 환경 변수 설정 (`.env`)
프로젝트 루트 또는 `data-pipeline` 폴더에 `.env` 파일을 생성하고 아래 변수를 입력합니다.
```env
# Gemini API Key (2단계 AI 분석용)
GEMINI_API_KEY=your_gemini_api_key_here
GEMINI_MODEL=gemini-2.5-flash # 또는 사용하고자 하는 모델명

# Spring Boot 백엔드 설정 (3단계 DB 적재용)
BACKEND_URL=http://localhost:8080
ADMIN_SECRET_KEY=your_spring_admin_secret_key
```

## 🗂️ 파이프라인 3단계 코어 명령어

파이프라인은 크게 **수집(Scrape) ➡️ 분석(Analyze) ➡️ 적재(Load)**의 3단계로 명확하게 나누어 동작해요. 각 단계는 독립적으로 실행할 수 있도록 유연하게 설계되었습니다. (`--scrape`, `--analyze`, `--load` 중 **하나를 선택**해서 실행해 주세요.)

### 1단계: 장소 및 방문자 리뷰 수집 (`--scrape`)
Playwright 브라우저를 백그라운드에서 실행해 실시간으로 검색어의 주소, 공식 이미지, 그리고 영수증 인증을 거친 진짜 방문자 리뷰를 수집해요. 수집된 결과는 `raw_places_YYYY-MM-DD.json` 파일로 안전하게 저장됩니다.
* **기본 명령**:
  ```bash
  python main.py --scrape --query "성수 조용한카페" --limit 5 --gui
  ```

### 2단계: Gemini AI 감성 및 카테고리 분석 (`--analyze`)
1단계에서 모은 데이터를 기반으로 Gemini API가 실제 방문자 리뷰의 깊은 맥락을 읽어내요. 공간의 분위기를 보여주는 무드 태그, 방문 동반자 유형, 한 줄 코멘트 등을 인공지능이 분석해 `analyzed_places_YYYY-MM-DD.json` 파일로 정밀하게 빌드합니다.
* **기본 명령**:
  ```bash
  python main.py --analyze --gui
  ```

### 3단계: 백엔드 DB 벌크 적재 (`--load`)
모든 AI 분석이 완료된 데이터를 읽어 Spring Boot 백엔드 서버의 벌크 적재 API로 한 번에 안전하게 쏙 주입(Ingest)합니다.
* **기본 명령**:
  ```bash
  python main.py --load
  ```

## 🖥️ 대화형 GUI 모니터 기능 지원 (`--gui`)

수집이나 분석을 실행할 때 `--gui` 플래그를 더해보세요. 진행 상황을 실시간으로 확인하고 제어할 수 있는 똑똑한 대화형 창이 나타납니다.

1. **눈으로 보는 진행 상태**: 전체 쿼리 중 현재 어느 단계를 지나고 있는지 진행률 게이지와 실시간 로그로 쉽게 확인해요.
2. **똑똑한 매칭 선택 모드**:
   - 특정 장소를 타겟으로 정밀 수집할 때(`--query`에 하이픈 `-` 포함), 네이버 검색 결과 1순위가 모호하거나 다른 매장일 경우 대화형 선택창을 띄워요.
   - 사용자는 GUI 화면에서 여러 후보 매장의 이름, 업종, 주소를 직접 대조한 뒤 진짜 매장을 **마우스 클릭 한 번으로 선택**할 수 있어요. 오매칭(예: 진짜 음식점 대신 입구 버스정류장이 수집되는 현상)을 원천 차단해 줍니다.
   - 수집이 불필요하다고 생각되는 장소는 `건너뛰기` 버튼으로 가볍게 스킵할 수도 있어요.

## ⚙️ 상세 명령 인자(Argument) 가이드

`python main.py`를 실행할 때 취향에 맞게 조합할 수 있는 다양한 옵션들이 준비되어 있어요.

| 옵션명 | 타입 | 기본값 | 설명 |
| :--- | :--- | :--- | :--- |
| **`--scrape`** | 플래그 | - | 네이버 지도 크롤링 단계 실행 (로컬 raw JSON 저장) |
| **`--analyze`** | 플래그 | - | Gemini AI 감성 분석 단계 실행 (로컬 analyzed JSON 저장) |
| **`--load`** | 플래그 | - | 분석이 끝난 JSON 데이터를 스프링 백엔드 DB에 적재 |
| **`--query`** | 문자열 | `합정 카페` | 수집할 키워드. 쉼표(`,`)를 이용해 여러 개 지정 가능.<br>특정 장소 타겟 수집 시 `지역 - 상호명` 형태로 입력 (예: `한남동 - 한남물터`) |
| **`--limit`** | 정수 | `3` | 검색어 1개당 수집할 최대 장소 개수 |
| **`--region`** | 문자열 | - | 단일 지역 필터명 (예: `강릉`, `송도`) |
| **`--category`** | 문자열 | - | 수집 대상 카테고리 필터 (`맛집`, `술집`, `카페`, `핫플레이스`, `디저트`, `명소` 중 선택) |
| **`--query-all`** | 플래그 | - | `regions.json`에 기재된 모든 지역과 6대 카테고리 키워드를 조합하여 대량 순회 수집을 진행 |
| **`--file`** | 문자열 | - | 저장/로드할 파일명 또는 날짜(`YYYY-MM-DD`).<br>미지정 시 오늘 날짜로 파일명이 결정되며, `--analyze` 또는 `--load` 시에는 디렉토리 내 가장 최근 파일이 자동 지정됨 |
| **`--delay`** | 실수 | `5.0` | 봇 차단 우회를 위한 검색 쿼리 간 대기 시간 (초) |
| **`--delay-random`** | 플래그 | - | 활성화 시 대기 시간이 `delay` ~ `delay * 2.5` 사이로 랜덤하게 지정되어 차단 우회력을 극대화 |
| **`--gui`** | 플래그 | - | 진행률 모니터 팝업 활성화 및 매장 모호성 발생 시 선택창 띄움 |

## 💡 유용한 실행 예시

### 1. 특정 키워드나 테마로 인기 장소들을 모으고 싶을 때 (기본 검색)
하이픈(`-`) 없이 키워드만 입력하면, 지정한 개수 한도(`--limit`) 내에서 현재 상위 노출되어 있는 매력적인 공간들을 자연스럽게 다수 수집합니다.
```bash
# '성수 조용한카페'와 '남양주 가볼만한 곳' 키워드로 각각 상위 5개씩 수집
python main.py --scrape --query "성수 조용한카페, 남양주 가볼만한 곳" --limit 5 --gui
```

### 2. 가보고 싶은 특정 매장들을 콕 집어 수집하고 싶을 때 (타겟 검색)
검색어에 하이픈(`-`)을 넣어 `지역 - 상호명` 형태로 요청하면, 정확한 주소와 상호명을 크로스 체크해 진짜 매장만 정밀하게 타겟팅해 수집해요. 매칭이 모호할 때는 GUI 창이 열려 선택을 도와줍니다.
```bash
# 특정 식당/카페 4곳을 타겟으로 각각 정확히 1개씩만 매칭하여 수집
python main.py --scrape --query "한남동 - 한남물터, 광교 - 심야식당 후라후라, 대전 - 백송한우 대전 유성, 강릉 - 강릉짬뽕순두부 동화가든 본점" --limit 1 --gui
```

### 3. 전국 단위의 대량 공간 데이터를 한 번에 수집하고 싶을 때 (대량 검색)
차단을 안전하게 피할 수 있도록 랜덤 대기 옵션(`--delay-random`)을 켜고 `regions.json`에 정의된 전체 리스트를 순회합니다.
```bash
python main.py --scrape --query-all --delay 7.0 --delay-random --gui
```

### 4. 네트워크나 API 오류로 수집이 중간에 끊겼을 때 (이어서 작업하기)
수집기와 분석기 모두 똑똑한 **실시간 머지(Merge) 및 이어하기(Resume) 기능**을 갖추고 있어요. 파이프라인을 다시 실행하면 이미 성공적으로 저장된 장소나 분석이 완료된 항목은 알아서 스킵하기 때문에, 네트워크 오류가 나더라도 아까운 AI API 비용과 시간을 아낄 수 있습니다.
```bash
# 가장 최신 수집본에서 아직 분석이 안 된 데이터만 이어서 Gemini AI 분석
python main.py --analyze --gui
```

## 🔍 수집 데이터 품질 필터링 정책

PickPl의 수집 엔진([scraper/portal_scraper.py](file:///c:/Users/Silok/Downloads/myProject/pickpl/data-pipeline/scraper/portal_scraper.py))은 깨끗하고 신뢰할 수 있는 데이터 퀄리티를 유지하기 위해 다음 필터링 규칙을 꼼꼼하게 적용하고 있어요:

1. **랜드마크/버스정류장 오매칭 차단**:
   - 상호명 끝에 `"입구"`, `"삼거리"`, `"정류소"` 같은 키워드가 들어가 타겟과 혼동을 줄 수 있는 랜드마크 데이터는 매칭 검증을 통해 똑똑하게 걸러내요.
2. **진성 영수증 방문자 리뷰 타겟팅**:
   - 네이버 플레이스 구조 변화에 따라 불필요하게 묻어 나오는 예약 링크, 혜택 안내 등 광고성 노이즈를 방지하기 위해 실제 방문자 리뷰 텍스트 클래스(`.zPfyi`, `.LwUKe`)만 골라서 깨끗하게 가져옵니다.
3. **스팸/블로그 광고글 배제**:
   - 리뷰 속에 `"체험단"`, `"무료숙박"`, `"제공받아 작성"`, `"협찬"` 같이 대가성 광고 뉘앙스를 풍기는 단어들이 포착되면 진성 리뷰에서 과감히 제외합니다.
4. **사장님 답글(댓글) 단독 제외**:
   - 손님 리뷰 아래에 달린 사장님의 감사 댓글이 섞여 들어가지 않도록, 인사말 패턴과 매장명 키워드를 분석해 사장님 답글은 한 번 더 안전하게 걸러냅니다.
