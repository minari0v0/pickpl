# Git Commit Rules

에이전트가 이 프로젝트에서 커밋을 작성할 때 반드시 준수해야 하는 규칙입니다.

1. **기능/분기 및 커밋 타입별 분리**
   - 커밋은 기능이나 분기 단위로 명확히 분리하여 수행하며, 알맞은 타입(`feat`, `fix`, `design`, `refactor`, `chore`, `docs` 등)을 명시합니다.

2. **극도로 간결하고 직관적인 커밋 제목**
   - 제목(첫 줄)이 장황해지는 것을 지양하고, 한눈에 파악할 수 있도록 아주 짧게 작성합니다.
   - *올바른 예시*: 
     - `design(explore): 검색탭 필터 및 그리드 레이아웃 적용`
     - `design(explore): 검색탭 리디자인`
   - *나쁜 예시*: 
     - `design(explore): 공간 탐색 탭 탭 필터 적용 및 유튜브식 그리드 레이아웃 리디자인`

3. **요약된 본문 작성**
   - 본문 내용은 디테일한 수정 항목을 너무 장황하게 나열하지 않고 핵심 위주로 깔끔하게 요약하여 작성합니다.
   - 단, 중요한 아키텍처 변경이나 주의사항 등 핵심 정보가 있다면 예외적으로 조금 자세히 서술할 수 있습니다.
