## 1. 프로젝트 개요
---
### 카테고리
> 라이프스타일 / 일상

### 한 줄 정의
PickPl 3.0 - 공간을 쇼핑하다: AI 기반 무드 큐레이션 공간 룩북 플랫폼

### 배경
사용자들은 데이트나 모임 장소를 찾을 때 수십 개의 블로그 텍스트 리뷰를 뒤지는 데 지쳐있습니다. 기존 지도 앱은 거리와 별점 위주로 공간을 보여주어, '조명이 어두운지', '의자가 푹신한지', '우드톤인지' 등 공간의 무드와 취향을 직관적으로 파악하기 어렵다는 문제점을 해결하고자 합니다.

### 사용자 문제
기존의 지도 앱과 텍스트 위주의 정보로는 '공간의 무드와 인테리어'를 직관적으로 파악하기 어렵다는 점이 가장 큰 사용자 문제입니다. 이는 사용자가 자신의 취향에 맞는 공간을 찾는 데 많은 시간과 노력을 들이게 합니다.

### 해결 방식
텍스트 리뷰를 과감히 버리고, 고화질 사진과 AI가 자동 생성한 무드 태그를 통해 공간을 직관적으로 '쇼핑'하게 만듭니다. 인스타그램 릴스나 틱톡처럼 세로로 스와이프하며 고화질 공간 사진을 넘겨보는 몰입형 룩북 피드를 제공하고, 다중 태그 조합으로 완벽한 공간을 0.1초 만에 검색하는 초정밀 필터링 기능을 제공하여 초개인화된 맞춤 공간 탐색 경험을 선사합니다.

### 차별점
경쟁 서비스들이 텍스트 리뷰와 평점 기반의 정보 제공에 머무는 반면, 'PickPl 3.0'은 무신사나 29CM 같은 패션 플랫폼처럼 '고화질 사진과 AI 무드 태그'를 핵심으로 하여 공간을 직관적으로 '쇼핑'하게 만듭니다. 특히 AI 자동화 파이프라인을 통해 유저의 수동 입력 없이 양질의 데이터를 무한으로 생산하여 '빈 화면(Cold Start)' 문제를 해결하고, 무자본 풀스택 배포 아키텍처를 통해 효율적인 운영을 가능하게 합니다.

### 타겟 사용자
특정 분위기의 공간을 찾는 비즈니스 사용자, 혼자만의 시간을 보낼 감성적인 공간을 찾는 솔로 사용자, 데이트나 모임 장소를 찾는 20~30대 커플 및 친구들 등 데이트나 모임 장소를 찾을 때 수십 개의 블로그 텍스트 리뷰를 뒤지는 데 지친 일반 대중입니다.

### 사용 시나리오
사용자는 데이트나 친구와의 모임 전, 혹은 혼자만의 시간을 보낼 아늑한 공간을 찾을 때 'PickPl 3.0'을 가장 먼저 떠올리고 사용하게 될 것입니다. 예를 들어, '비오는 날 + 플랜테리어 + 콘센트석'과 같이 특정 무드와 필요한 시설을 조합하여 원하는 공간을 빠르게 탐색합니다.

### 제품 목표
기존 지도 앱과 텍스트 리뷰의 한계를 넘어, '공간의 무드와 취향'을 직관적으로 탐색하고 발견할 수 있도록 하여 사용자에게 초정밀 개인 맞춤 공간 검색 경험을 제공하고, 무드 기반 공간 검색 시장을 개척합니다.

### 핵심 KPI
월간 활성 사용자(MAU) 증가율, 평균 세션 시간, 컬렉션 보드 저장 수, 멀티 태그 검색 기능 사용률, AI 태그 정확도(내부 지표), 서비스 이탈률 감소.

### 리스크/이슈
AI 기반 무드 태깅의 정확도 및 일관성 부족, 사용자에게 새로운 공간 탐색 방식에 대한 학습 요구, 기존 지도/검색 서비스의 강력한 시장 지배력, 데이터 수집 API의 정책 변화 가능성, 무료 클라우드 인스턴스의 안정성 및 스케일업 한계.

## 1. 몰입형 공간 룩북 피드
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

인스타그램 릴스나 틱톡처럼 세로로 스와이프하며 고화질 공간 사진을 넘겨보는 480px 몰입형 룩북 피드를 제공하여, 사용자가 공간의 분위기를 직관적으로 경험하고 '쇼핑'하듯이 탐색할 수 있도록 합니다.

**수용 기준:**
```
1. ○ 피드 화면에서 세로 스와이프 시 다음 공간의 고화질 사진이 부드럽게 전환되어야 한다.
2. ○ 각 공간 사진은 480px 이상 고해상도로 표시되어야 한다.
3. ○ 각 공간 사진에는 공간명, 간단한 무드 태그 등이 함께 표시되어야 한다.
4. ○ 피드 내에서 '하트' 버튼을 눌러 해당 공간을 컬렉션 보드에 저장할 수 있어야 한다.
```


---


## 2. 초정밀 취향 맞춤 필터링 검색
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

[#비오는날] + [#플랜테리어] + [#콘센트석] 등 다중 태그 조합으로 사용자가 원하는 완벽한 공간을 0.1초 만에 검색할 수 있도록 초정밀 필터링 기능을 제공합니다. 이를 통해 사용자의 특정 요구사항을 정확하게 반영한 공간을 신속하게 찾아줍니다.

**수용 기준:**
```
1. ○ 최소 3개 이상의 AI 무드 및 시설 태그를 조합하여 검색할 수 있어야 한다.
2. ○ 다중 태그 검색 결과가 0.1초 이내에 사용자에게 표시되어야 한다 (Redis 캐싱 활용).
3. ○ 검색 결과는 사용자가 선택한 태그와 높은 연관성을 가져야 한다.
4. ○ 태그 자동 완성 및 추천 기능을 제공하여 검색 편의성을 높여야 한다.
```

## 3. 개인 컬렉션 보드
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전

피드에서 마음에 드는 공간에 딥 말차색 '하트' 버튼을 누르면, 핀터레스트 형태의 카드로 예쁘게 적재되는 개인 컬렉션 보드를 제공합니다. 유저의 최근 활동 로그를 트래킹하여 맞춤형 추천을 위한 데이터를 축적하고, 저장된 공간들을 한눈에 확인하고 관리할 수 있도록 합니다.

**수용 기준:**
```
1. ○ 하트 버튼 클릭 시 해당 공간이 컬렉션 보드에 카드 형태로 저장되어야 한다.
2. ○ 컬렉션 보드에서 저장된 공간들을 핀터레스트 스타일로 시각적으로 쉽게 탐색할 수 있어야 한다.
3. ○ 컬렉션 보드 내 저장된 공간을 삭제하거나 추가할 수 있는 기능을 제공해야 한다.
4. ○ 사용자의 컬렉션 저장 활동 로그가 정확히 트래킹되어야 한다.
```

## 4. AI 기반 공간 정보 자동 분석 및 태깅
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자 입력 없이 양질의 데이터를 무한으로 생산하기 위해, Spring Batch를 이용해 카카오/네이버 API에서 핫플레이스의 사진 URL과 리뷰 텍스트를 크롤링하고, Gemini 1.5 Flash API를 활용하여 사진과 텍스트를 멀티모달 방식으로 분석하여 공간의 분위기 및 시설을 자동으로 태깅하고 DB에 저장합니다.

**수용 기준:**
```
1. ○ 카카오/네이버 API를 통해 핫플레이스의 사진 URL과 리뷰 텍스트가 주기적으로 자동으로 크롤링되어야 한다.
2. ○ 수집된 사진과 텍스트 데이터가 Gemini 1.5 Flash API를 통해 성공적으로 분석되어야 한다.
3. ○ AI가 공간의 분위기와 시설을 파악하여 {'mood': ['우드톤', '코지함'], 'facility': ['노트북작업']} 형태의 JSON으로 정확하게 반환해야 한다.
4. ○ AI가 반환한 태그 정보가 백엔드 DB에 자동으로 매핑 및 저장되어야 한다.
5. ○ AI 태깅된 데이터가 서비스 프론트엔드에서 정상적으로 검색 및 필터링에 활용되어야 한다.
```

## 5. 서비스 성능 및 확장성 확보
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전

초기 자본 0원으로 대규모 트래픽을 방어하고 안정적인 서비스를 제공하기 위해, Vue.js(또는 Next.js) 기반의 프론트엔드를 Vercel에 배포하고, Spring Boot 기반 백엔드 및 MySQL, Redis를 Oracle Cloud ARM 인스턴스에 Docker Compose로 배포하여 시스템의 안정성과 확장성을 확보합니다.

**수용 기준:**
```
1. ○ 프론트엔드가 Vercel을 통해 글로벌 CDN 환경에서 안정적으로 제공되어야 한다.
2. ○ 고화질 이미지 로딩 시 트래픽 비용 없이 빠른 응답 속도를 보여야 한다 (원본 URL Hotlinking 활용).
3. ○ 백엔드 API 호출 시 평균 응답 시간이 500ms 이내여야 한다.
4. ○ Redis 캐싱을 통해 인기 피드 및 태그 검색 속도가 최적화되어야 한다.
5. ○ Oracle Cloud ARM 인스턴스 내 Docker Compose 환경에서 Spring, MySQL, Redis 컨테이너가 안정적으로 운영되어야 한다.
```
